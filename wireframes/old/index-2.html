What Is Cameo Runtime?

The Unbrowser for Apps, Not Sites

Browsers restrict what HTML5, CSS and JavaScript can do, to protect security and privacy.  But an HTML5 app needs access to platform features. Cameo is for apps, not sites.

High Performance, High Compliance

Each Cameo runtime is tuned to the OS it runs on.  Cameo is based on Chromium/Blink technology and the latest W3C APIs for great performance and exemplary standards compliance.

Runs Where No Other Dares to Go

Cameo is aimed at Tizen Linux, Windows (desktop), Android and Mac OS X.  It can be Embedded in each app, or be built into a Managed platform to be shared among multiple apps.

Why Use Cameo Runtime?

The Features You Need, Without the Overhead You Don’t

Browsers clutter your UI. A browser can’t allow your app access to features a site might maliciously exploit.  With Cameo your app has access to the features you need – safely.

Lock in Features, Extend with Plug-ins

Eliminate compatibility worries by packaging your app with Cameo.  No one can update your app’s foundation except you.  Use Cameo’s plug-in interface to add APIs unique to your application.

HTML5 Apps for your Platform

Building a platform?  Bundle the Cameo Managed runtime and bring the benefits of Cameo to your application ecosystem without the need for each app to include it.

How Does it Work?

Tailored for the needs of Apps

A simple native API invokes Cameo passing in your application’s HTML5, CSS and JavaScript.  Cameo plug-ins let you add unique features without compromising browser security.

Managed for You

The Cameo Managed runtime environment handles the entire application life cycle from invocation to termination.  Pick the OS you want for your platform, while offering the HTML5 features developers demand.

The Best of Chromium, Remixed

Cameo remixes the best of Chromium and Blink to meet the unique needs of apps written in HTML5, CSS and JavaScript.  The browser bits are left behind.
